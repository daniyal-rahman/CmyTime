# Project CmyTime: A C. elegans Simulation for Emergent Behavior

## 1. High-Level Goal

The primary goal of this project is to simulate the nematode *Caenorhabditis elegans* (C. elegans) to study how complex, goal-directed behaviors can emerge from a known biological neural circuit (a connectome). We aim to observe how a simulated worm, endowed with a nervous system based on real biological data, can learn to navigate its environment, seek food, and avoid danger through a process of reward-based synaptic plasticity.

This serves as a computational sandbox to explore fundamental questions in neuroscience:
*   How does the structure of a neural network dictate its function?
*   Can simple, local learning rules give rise to complex, intelligent behavior?
*   To what extent can we model biological learning and decision-making in silico?

## 2. Core Concepts

The simulation is built on several key principles from computational neuroscience and artificial life.

### 2.1. Agent-Based Modeling
The simulation is structured around an **Agent**, which represents the C. elegans worm. This agent is an autonomous entity that perceives its environment, makes decisions using its "brain," and acts upon the environment. This bottom-up approach allows us to observe how the interactions between the agent and its environment lead to complex system-wide behavior.

### 2.2. Biologically-Plausible Movement
A cornerstone of this project is the realistic modeling of C. elegans locomotion. Instead of using abstract commands like "turn left" or "move forward," the agent's movement is an emergent property of its motor system.
*   **Dorsal-Ventral Muscle Groups:** The model simulates the two primary muscle groups that run along the worm's body: dorsal and ventral.
*   **Differential Activation:** The agent moves by differentially "activating" these two muscle groups.
    *   **Turning:** Turning is achieved by creating an imbalance in activation. A stronger dorsal activation and weaker ventral activation results in a turn in one direction, and vice-versa. This mimics how the real worm bends its body.
    *   **Forward Movement:** Forward propulsion is driven by the sum of the activations of both muscle groups, simulating the propulsive force generated by the worm's characteristic sinusoidal body wave.

### 2.3. Emergent Behavior
We do not explicitly program the worm to "navigate" or "avoid traps." Instead, we provide it with a nervous system, a body, and a learning rule. The desired behaviors—such as chemotaxis (moving towards a chemical stimulus) and trap avoidance—are intended to **emerge** from the interaction between the agent's learning process and the environmental feedback.

### 2.4. Spike-Timing-Dependent Plasticity (STDP)
STDP is the primary learning mechanism in the `LearningBrain`. It is a biologically-observed process where the precise timing of pre-synaptic and post-synaptic spikes determines how the strength of the connection between them changes.
*   **"Fire together, wire together":** If a pre-synaptic neuron fires just before a post-synaptic neuron (and thus likely contributed to its firing), the synapse is strengthened (Long-Term Potentiation).
*   **"Out of sync, lose the link":** If the pre-synaptic neuron fires just after the post-synaptic neuron, the synapse is weakened (Long-Term Depression).
In our model, this process is **neuromodulated by reward**. The changes suggested by STDP are only applied if they are consistent with the outcome of the action. Positive rewards (e.g., getting closer to food) "confirm" the recent synaptic changes, while negative rewards (e.g., hitting a trap) suppress or reverse them.

## 3. System Architecture

The codebase is designed to be modular and extensible, with a clear separation of concerns.

### 3.1. `Agent` (`src/core/agent.py`)
This is the central class, encapsulating the agent's `brain` and `body`. It is responsible for receiving an observation from the environment, passing it to the brain to get an action, and returning that action to the environment.

### 3.2. `Brain` (`src/brains/`)
The brain is the decision-making center. There are two types:
*   **`StaticBrain`**: This brain has a fixed connectome. Its synaptic weights do not change. It serves as a baseline to understand the innate, "instinctual" behavior of the C. elegans connectome. To make it functional, a **turning bias** is hard-coded, which adds a small directional input to the motor neurons based on the angle to the stimulus. This provides a basic navigational "scaffold."
*   **`LearningBrain`**: This brain has a plastic connectome. It uses the STDP learning rule described above to modify its synaptic weights based on reward signals from the environment. It also uses the turning bias as an initial scaffold, but the goal is for the STDP mechanism to learn to refine and eventually override this simple instinct with more sophisticated, learned behaviors.

### 3.3. `Environment` (`src/environments/`)
The environments are compliant with the Gymnasium API. They define the world, its physics, and the reward structure.
*   **Action Space:** The action space for all environments is a 2-element array representing `[dorsal_activation, ventral_activation]`.
*   **Observation Space:** The agent observes its position, the stimulus location, the relative angle and distance to the stimulus, and a "gradient" vector pointing towards the stimulus. In the `TrappedChemoattractionEnv`, it also observes the location of the nearest trap.
*   **Reward Function:** The primary reward is based on the change in distance to the food source (closer is better). The `TrappedChemoattractionEnv` adds a large negative penalty for entering a trap zone.

### 3.4. `LIFNeuron` (`src/models/lif_neuron.py`)
The `LearningBrain` is composed of Leaky Integrate-and-Fire (LIF) neurons. This is a simple but powerful spiking neuron model.
*   **Membrane Potential:** Each neuron has a `membrane_potential` that "leaks" back to a resting state over time.
*   **Input Current:** Incoming signals from other neurons (or sensory input) are treated as `input_current`, which increases the membrane potential.
*   **Spiking:** When the potential reaches a `threshold`, the neuron "fires" a spike, and its potential is reset to a `reset_potential`. This spike is then propagated to other neurons.

### 3.5. `Simulator` (`src/core/simulator.py`)
This class orchestrates the main simulation loop. For a given number of steps, it:
1.  Gets an action from the agent.
2.  Passes the action to the environment and gets the next observation and reward.
3.  If the agent has a learning brain, it calls the brain's `update_weights` method with the reward.
4.  Records data for later visualization.

## 4. Data: The C. elegans Connectome

The simulation uses real biological data from the OpenWorm project (`NeuronConnectFormatted.xlsx` and `CElegansNeuronTables.xls`). This data provides:
*   **The Connectome:** A list of all identified neurons in the C. elegans nervous system and the synaptic connections between them (including the number of synapses, which we use as a proxy for weight).
*   **Neuron Types:** Categorization of neurons into sensory, motor, and interneuron groups. We further process this to identify the specific dorsal and ventral motor neurons required for our movement model.

## 5. Simulation Workflow & Output

The simulation is run via `src/main.py`. Command-line arguments allow for selecting the brain (`--brain`), environment (`--env`), and number of epochs/steps.

The script generates two primary outputs in `results/figures/`:
1.  **Trajectory GIF (`..._trajectory.gif`):** An animation showing the agent's trajectory for each epoch. This is useful for visualizing the learning process frame-by-frame.
2.  **Gradient Plot (`..._all_epochs_trajectory_gradient.png`):** A static plot showing all epoch trajectories overlaid, with a color gradient from early epochs (blue/purple) to later epochs (yellow). This provides a clear, at-a-glance view of how the agent's behavior changes over time. An efficient, successful agent will show trajectories that become shorter and more direct towards the food source while consistently avoiding traps in later epochs.

## 6. Future Scope

The modular design of this project provides a strong foundation for future expansion and research. Potential future directions include:

*   **More Complex Environments:** Developing new environments with different challenges, such as mazes, moving obstacles, or competitive multi-agent scenarios to test the limits of the agent's learning capabilities.
*   **Exploring Other Connectomes:** Adapting the simulation to use the connectome of other, more complex organisms, such as the *Drosophila* (fruit fly). This would allow for comparative studies of how different neural architectures solve similar problems.
*   **Integration with Modern Reinforcement Learning:** Combining the biologically-constrained model with state-of-the-art machine learning. This could involve using deep reinforcement learning algorithms to train an agent with a similar body and action space, providing a benchmark to compare the performance and efficiency of biological learning rules against engineered ones.
